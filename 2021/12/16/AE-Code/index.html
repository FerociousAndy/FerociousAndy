写表达式时注意输入法的中英文字符和符号的使用，包括 () ‘’ , 等英文符号。备注：内容不跟随视频内容，表达式后面的大写单词只是代表数值和缩写无关。

1.

  • timeN 一般应用角度和度数相关。例如time120：每秒钟物体旋转n=120度。

  • time*(N/M)表示每M秒旋转N度，可以用来降低旋转速度。

  • time在位置P上面移动：value+time*N 尝试建议将p的属性分开，分离XY，单独在单一属性上面加表达式：

如果在P原属性的基础上，在y轴上面没有效果

N表示的是每秒移动的距离，正数+向右，负数-向左。

  • posterizetime（A）；value 视频中的value是time的属性值，具体为timeN或者time（N/M），在p的属性下元素是跳动移动位置。

其中A表示每秒跳几步，如果是3，一秒内跳3步；在p的属性下，N表示总距离数，M表示完成N的距离数需要跳几秒。

比如：posterzetime（2）；time*1000 表示一秒跳1000个单位距离，需要跳两步。

​ posterzetime（3）；time*（500/10）表示10秒跳500个单位距离，每秒跳3步，共需要30步。

如果是R的属性下表示的则是旋转度数的跳动变化。

如果你的元素没有完成指定的N的行为，请检查自己时间是不是符合M的范围

2.

常用 wiggle(N,M) N表示每秒震动频率，震动的次数，M 表示每次震动时偏离原位置的像素值或振幅 amplitude。

wiggle(10,5)表示该属性下的元素每秒震动10次，每次偏离5个点之间（+5到-5）。

wiggle(freq, amp, octaves = 1, amp_mult = .5, t = time)

freq：振频，即每秒抖动次数 ,没有默认值，必须传参

amp：振幅，即增大或减少的幅度 , 没有默认值，必须传参

octaves：在基波上增加谐波的量，有效参数为0到10之间的整数 ，即使输入浮点数也会被向上取整，默认数值为1

amp_mult：谐波放大的量，作为放大器使用，如果octaves小于等于1，则该参数传入多少都不会产生变化，默认数值为0.5

t：采样时间，如果传入的是个固定的数值，则持续返回这个数值对应的采样时间下的结果，抖动就失去意义，默认传入time[1]函数

以上都是非等比变化

等比的变化：

s=wiggle(N,M)[0];[s,s] 次数s是附值，可以使用其他的参数代替，保证s后面的[s,s]是同一个字符。

其中[0],0为第一个数组，1为第二个数组（具体可以参考链接：http://www.video7cg.com/26201.html）

3.

loopin(“style”,N)；loopout(“style”,N) 一般用在位置相关的地方，style包括cycle/pingpong/continue/offset，N可以等于0，但是小于关键帧总数。looping和loopout默认的样式可以参照属性里面的格式。

loopin表示保持style运动在关键帧前进行运动。

loopout表示保持style运动在关键帧后进行运动。

cycle表示循环运动

pingpong表示往复运动

offset表示按照N的方式延续运动下去

0的时候，表示所有关键帧循环；1的时候，表示只循环最后两帧；以此类推。

continue表示在保持最后一帧的运动方式继续运动下去，备注continue不存在后面N的参数，会报错。

4.

value

value+N：P的位置分离单一的属性中，首先现有位移关键帧动画，这个表示的是在原有关键帧位移上面有N的偏移，N为正数，x轴偏右，y轴偏下；N为负数，x轴偏左，y轴偏上，偏移值为N的绝对值的单位数。

valueAtTime

位置延迟跟随，第一个元素跟随第二个元素，创建连接，会在元素二创建表达式前缀入：thisComp.layer(“第一个元素”).transform.yPosition 第一个元素为元素名称，y为所连接的轴向。

thisComp.layer(“第一个元素”).transform.yPosition.valueAtTime(time-N)

-N表示第二个元素在第一个后面出现，时间为第一个元素运动N秒推迟后，才开始运动。

如果为+N则会出现在第一个元素上面。

5.数字随着物体位置移动产生对应的变化，文本position位置属性链接/速度属性

position[0]/positon[1]/.speed/.speed.fixto[N]

将数字的source源文本 进行表达式，并且将链接到某物体的位置属性上，自动产生表达式链接。

如：thisComp.layer(“物体名称”).transform.position[0]

备注个人理解：0表示非正常显示的状态，似乎只支持0和1的数字。

thisComp.layer(“物体名称”).transform.position[1]

变成1的话，数字显示为该物体在p运动方向，比如y轴上面的位置数值变化。可以从ctrl+r的标尺查看。

文本展示速度属性.speed/.speed.tofix[N]

thisComp.layer(“物体名称”).transform.position.speed

删除[0]在后面添加.speed表示：文本数值表示的是物体的速度值，包括小数点后面的位数，具体值可以查看速度曲线。

.speed.tofix[N] fix把速度值固定N表示的是小数点后面的位数

例如thisComp.layer(“物体名称”).transform.position.speed.tofix[2]，速度值固定到小数点后面两位数。

speed和velocity区别：speed不带有方向，而velocity带有方向性。

以文字文本表达式链接物体，同speed

在后面输入.velocity,表示物体的速度属性，并且有区分xyz三轴，如果某一轴上面有速度，则会显示速度值。

.velocity[N]，N可以取值0，1，2 0表示显示x轴属性值，1表示显示y轴属性值，3表示显示z轴属性值，相对的隐藏其他两种属性值，与各个值是不是有值无关。

例如thisComp.layer(“物体名称”).transform.position.velocity[1] 并且文本显示0.113216543

则为：该物体在此时刻上的y轴速度值为0.113216543

并且会存在负数值，比如.velocity[1] -013，物体y轴向上移动为负数值，若改为正数值，如：.velocity[1]*-1,则会变为正值，并且一个-1可以作用于三个xyz轴的负值。

6.线性表达式

linear线性

linear(t, Min, Max, value1, value2)表示（个人理解）：

time关键帧的时间

如果完成整个运动从value1到value2的位置value数值所需要走的步数

min从第几步开始走

max完成整个运动需要走的步数

value是指的是物体完成运动的位置数值

value1开始变化的位置数值

value2结束变化的位置数值

位置数值，上负数，下正数

视频中是创建了空物体＋表达式滑块控制，进行属性的链接到空物体，并且视频中的value这英文单词可以直接进行省略。

例如 linear(thisComp.layer(“kongzhiqi “).effect(“滑块控制”)(“滑块”),0,100,-200,1000)

滑块控制，运动物体从-200位置移动到1000的位置，需要走100步，完成的关键帧时间为(thisComp.layer(“kongzhiqi “).effect(“滑块控制”)(“滑块”)上面的关键帧。

linear(t, value1, value2)表示当time在0到1之间时，从value1变化到value2；

视频中说其他包括easeIn easeOut 等的用户只需要替换linear就可以了，并且注意大写的I和O字母，输入会报错，注意自己的最后的括号。
